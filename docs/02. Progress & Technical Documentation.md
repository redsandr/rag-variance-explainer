## What's Been Built

### Core Pipeline (11 modules)

| File | Function |
|------|----------|
| `src/config.py` | Centralized config from env vars — all thresholds, weights, model names configurable |
| `src/ingest.py` | SEC EDGAR fetcher: ticker→CIK, submissions, 10-K/10-Q listing, MD&A HTML extraction |

| `src/retrieval.py` | ChromaDB query + multi-strategy retrieval (bi-encoder, keyword boost, forward-looking penalty) |
| `src/cross_encoder.py` | Cross-encoder re-ranking (MiniLM-L-6) with hybrid scoring (CE 0.9 + BI 0.1 + penalty + boost) |

## Architecture & Data Flow

```
User Question
    │
    ▼
  query_expansion (5 synonyms, 20 financial groups)
    │
    ▼
  ChromaDB + nomic-embed (bi-encoder) ── top 20 candidates
    │
    ▼
  forward_looking_penalty (configurable patterns/weight)
    │
    ▼
  cross-encoder re-ranking (MiniLM-L-6, hybrid score)
    │
    ▼
  top_k chunks → LLMClient → grounded answer + citations
```

### Data Pipeline

```
SEC EDGAR ──→ ingest.py ──→ chunking.py ──→ embedding.py ──→ ChromaDB (740 chunks)
```

## Key Technical Decisions

| Decision | Rationale |
|----------|-----------|
| **Local LLM (llama.cpp)** | No ongoing API cost. Swappable to Anthropic/OpenAI via .env |
| **ChromaDB** | Local, file-based, zero infra — fits portfolio demo |
| **SEC EDGAR as data source** | Structured XBRL + narrative MD&A — no PDF parsing needed |
| **nomic-embed + prefixing** | search_document vs search_query prefixes required by model |
| **Cross-encoder re-ranking** | recall@1 improved 0.14 → 0.23 (+64%) |
| **Forward-looking penalty** | Solves risk-factor chunk pollution in MD&A retrieval |
| **Hybrid scoring** | 0.9 CE + 0.1 bi-encoder + configurable penalty + keyword boost |
| **Token-based chunking** | tiktoken cl100k_base, recursive split by paragraph→line→token |
| **Table row joining** | SEC HTML uses one \<td\> per token; rows are joined before extraction |
| **Qwen2.5-7B-Instruct Q4_K_M** (ex-VL) | Awal pake \`Qwen2.5-VL-7B-Instruct-Q4_K_M.gguf\` — VL variant wastes weight on vision unused in text pipeline. **Swapped to \`Qwen2.5-7B-Instruct-Q4_K_M.gguf\`** — full 7B capacity for text reasoning. +4.54pp faithfulness (69.7% → 74.24%). GPU via llama.cpp ✅. |
| **hashlib.md5 for dedup** | Python's built-in `hash()` is randomized per process — `hashlib` gives stable keys across runs |
| **`pending_question` pattern** | Streamlit forbids widget value changes after widget instantiation; set session_state before widget render |
| **Safe config defaults** | `cross_encoder_device` defaults to `"cpu"` — won't crash on machines without CUDA. Current `.env`: `cuda` (GPU aktif via PyTorch 2.11.0+cu128) |
| **seed=42 for deterministic output** | Tanpa seed, LLM generate output beda tiap run (eval-003 0%→100%) |
| **Period integrity prompt** | Dominant failure mode (71% period cross-contamination) — single rule #7 + 4-item checklist |
| **LLM singleton failure recovery** | Reset `_instance` kalau init gagal biar ga corrupted permanen |
| **UI error boundary** | Wrap RAG calls di try/except — `st.error()` instead of crash |
| **Answer caching in eval** | `faithfulness_results.json` sekarang include answer text + sources → re-export judge prompts tanpa model via `--from-cache` |

## Phase 7e — Faithfulness Fix — Phase 1 (Jul 2026)

Three targeted fixes across 3 pipeline layers to raise strict faithfulness from 65.8% → target 75-80%.

### Changes

| Change | Files | Impact |
|--------|-------|--------|
| **Number Transposition Fix** (1.1) | `src/rag.py` | Integrasi `verify_answer()` — append warning ke answer kalo ada angka yang gak match source |
| **Metric Conflation Fix** (1.2) | `src/post_process.py`, `src/rag.py` | `MetricVerifier` class — cross-check metric names in answer vs source using synonym groups |
| **Causal Proximity Fix** (1.3) | `src/build_index.py`, `src/retrieval.py`, `src/post_process.py` | `tag_chunk()` → enriched metadata (metric label), `_filter_by_metric()` → filter chunks before cross-encoder |

### Detail

**Number Transposition Fix (`0e364b4`):**
- `verify_answer()` yang sudah ada dipanggil setelah LLM generate

## Phase 7d — CI Debugging & `gh` CLI Setup (Jul 2026)

CI failed on commit `bfdf3a1` with exit code 2. Debugged via GitHub API (logs API 403 tanpa auth → installed `gh` CLI + fine-grained token).

### Root Cause

```python
# src/hybrid_search.py — simplified
try:
    from rank_bm25 import BM25Okapi      # ini conditional
    HAS_BM25_LIB = True
except ImportError:
    HAS_BM25_LIB = False                  # BM25Okapi TIDAK pernah di-assign

def build_bm25(corpus: list[str]) -> BM25Okapi | TfidfVectorizer:  # 💥 NameError!
```

Python evaluates type annotations eagerly at function definition time. **Kenapa lolos di local?** Karena `rank-bm25` sudah terinstall — import sukses, `BM25Okapi` terdefinisi. Bug laten yang cuma muncul di CI fresh environment.

### Fix

```python
from __future__ import annotations   # ← baris baru

import hashlib
import re
```

`from __future__ import annotations` mengubah semua annotation jadi lazy string — tidak dievaluasi saat module load. `BM25Okapi | TfidfVectorizer` disimpan sebagai string literal `"BM25Okapi | TfidfVectorizer"` dan tidak pernah di-resolve sampai benar-benar diakses (tidak pernah, karena hanya dipakai type checker).

## Phase 7f — Prompt Tuning & Parser Fix (Jul 17, 2026)

After Phase 7e (74.24% strict), two issues remained: judge parser failed on eval-003 ("22024" year typo → ERROR), and User env var `LLAMA_CPP_MODEL_PATH` kept overriding `.env`.

### Parser Fix: Parenthetical Explanations

**Root cause:** Judge LLM returned `"UNFAITHFUL" (explanation text)` — parenthetical notes after verdict strings, not `//` comments. The old `_strip_json_comments()` only handled `//` comments.

**Fix:** `_strip_json_comments()` → `_strip_json_trailing()` (`src/eval_faithfulness.py`):
- Handles both `//` inline comments AND `"VALUE" (extra text)` patterns
- Detects closing `"` after verdict value, checks if remaining text starts with `(` or `)`
- Removes trailing garbage before JSON parsing

### Judge Prompt Refinement

**Problem:** Judge prompt was verbose (full example section) but still produced extra-text outside JSON. Shortened version without examples caused under-extraction (1.6 claims/question vs 3.3 previously).

**Fix:** Restored examples in `JUDGE_SYSTEM_PROMPT_FULL` but added explicit instruction: `"Return ONLY valid JSON — no comments, no explanations, no text outside the JSON object."`

### Env Var Anomaly

**Root cause:** Windows User env var `LLAMA_CPP_MODEL_PATH` was set to old VL model path. `os.getenv()` reads this before `.env`, so `.env` changes had no effect.

**Fix:** `[Environment]::SetEnvironmentVariable("LLAMA_CPP_MODEL_PATH", $null, "User")` — permanent removal. Model now reads from `.env` correctly.
