## What's Been Built

### Core Pipeline (11 modules)

| File | Function |
|------|----------|
| `src/config.py` | Centralized config from env vars — all thresholds, weights, model names configurable |
| `src/ingest.py` | SEC EDGAR fetcher: ticker→CIK, submissions, 10-K/10-Q listing, MD&A HTML extraction |
| `src/chunking.py` | Token-based recursive splitting (tiktoken cl100k_base), structure-aware (paragraph→line→token) |
| `src/embedding.py` | nomic-embed-text-v1.5 (768-dim) with search_document/search_query prefixing |
| `src/build_index.py` | End-to-end pipeline: fetch → chunk → embed → store. 432 chunks indexed across 24 filings |
| `src/retrieval.py` | ChromaDB query + multi-strategy retrieval (bi-encoder, keyword boost, forward-looking penalty) |
| `src/cross_encoder.py` | Cross-encoder re-ranking (MiniLM-L-6) with hybrid scoring (CE 0.9 + BI 0.1 + penalty + boost) |
| `src/query_expansion.py` | Financial glossary-driven synonym expansion (20 groups, configurable n_terms) |
| `src/glossary.py` | Per-company XBRL tag mappings (CMG, DRI, CBRL) |
| `src/llm.py` | 3-backend LLM client (llama.cpp / Anthropic / OpenAI), singleton pattern, GPU offloading |
| `src/rag.py` | RAG orchestrator: retrieve → build context → generate → sourced answer with citations |

## Architecture & Data Flow

```
User Question
    │
    ▼
  query_expansion (5 synonyms, 20 financial groups)
    │
    ▼
  ChromaDB + nomic-embed (bi-encoder) ── top 20 candidates
    │
    ▼
  forward_looking_penalty (configurable patterns/weight)
    │
    ▼
  cross-encoder re-ranking (MiniLM-L-6, hybrid score)
    │
    ▼
  top_k chunks → LLMClient → grounded answer + citations
```

### Data Pipeline

```
SEC EDGAR ──→ ingest.py ──→ chunking.py ──→ embedding.py ──→ ChromaDB (740 chunks)
```

## Key Technical Decisions

| Decision | Rationale |
|----------|-----------|
| **Local LLM (llama.cpp)** | No ongoing API cost. Swappable to Anthropic/OpenAI via .env |
| **ChromaDB** | Local, file-based, zero infra — fits portfolio demo |
| **SEC EDGAR as data source** | Structured XBRL + narrative MD&A — no PDF parsing needed |
| **nomic-embed + prefixing** | search_document vs search_query prefixes required by model |
| **Cross-encoder re-ranking** | recall@1 improved 0.14 → 0.23 (+64%) |
| **Forward-looking penalty** | Solves risk-factor chunk pollution in MD&A retrieval |
| **Hybrid scoring** | 0.9 CE + 0.1 bi-encoder + configurable penalty + keyword boost |
| **Token-based chunking** | tiktoken cl100k_base, recursive split by paragraph→line→token |
| **Table row joining** | SEC HTML uses one \<td\> per token; rows are joined before extraction |
| **Qwen2.5-7B-Instruct Q4_K_M** (ex-VL) | Awal pake \`Qwen2.5-VL-7B-Instruct-Q4_K_M.gguf\` — VL variant wastes weight on vision unused in text pipeline. **Swapped to \`Qwen2.5-7B-Instruct-Q4_K_M.gguf\`** — full 7B capacity for text reasoning. +4.54pp faithfulness (69.7% → 74.24%). GPU via llama.cpp ✅. |
| **hashlib.md5 for dedup** | Python's built-in `hash()` is randomized per process — `hashlib` gives stable keys across runs |
| **`pending_question` pattern** | Streamlit forbids widget value changes after widget instantiation; set session_state before widget render |
| **Safe config defaults** | `cross_encoder_device` defaults to `"cpu"` — won't crash on machines without CUDA. Current `.env`: `cuda` (GPU aktif via PyTorch 2.11.0+cu128) |
| **seed=42 for deterministic output** | Tanpa seed, LLM generate output beda tiap run (eval-003 0%→100%) |
| **Period integrity prompt** | Dominant failure mode (71% period cross-contamination) — single rule #7 + 4-item checklist |
| **LLM singleton failure recovery** | Reset `_instance` kalau init gagal biar ga corrupted permanen |
| **UI error boundary** | Wrap RAG calls di try/except — `st.error()` instead of crash |
| **Answer caching in eval** | `faithfulness_results.json` sekarang include answer text + sources → re-export judge prompts tanpa model via `--from-cache` |

### Session 5 — Streamlit Bug Hunt & UX Overhaul (18 Jul 2026)

**Root cause**: Streamlit DOM isolation — HTML `<div>` di satu `st.markdown()` dan penutup di `st.markdown()` lain gak pernah nesting. Tiap `st.*` call punya container sendiri.

**Bugs fixed (7)**:
1. `name 'get_llm' is not defined` — function defined after `_startup_check()`. Fix: move `get_llm()` before `_startup_check()`.
2. `st.session_state.question_input` widget conflict — session_state set after widget creation. Fix: pop `pending_question` BEFORE widget, use `value=` param.
3. Example chips not filling input — Streamlit ignores `value=` after first render. Fix: set `st.session_state.question_input = pending_question` before widget.
4. Example chips ignoring checkbox state — timing: checkbox value not captured at click. Fix: save widget state (`_saved_compare`, `_saved_topk`, `_saved_ticker`) at click moment, read from `st.session_state.get()`.
5. Stale processing freeze — widget change during processing orphans model thread. Fix: `@st.fragment` wraps analyze logic + `st.form()` isolates widgets.
6. NameError `compare` — example buttons render before checkbox. Fix: read from `st.session_state.get()` instead of local var.
7. Broken layout — HTML grid across `st.markdown()` calls never nests. Fix: pure Streamlit `st.columns([3, 1])` + `st.columns(4)` for chips.

**Key learnings**:
- `st.form()` — widget changes inside form DON'T trigger rerun. Only `st.form_submit_button()` triggers.
- `@st.fragment` — long-running operations immune to widget change reruns.
- `disabled=processing` — requires processing set BEFORE widget render (two-pass: set → rerun → render).
- Example chips need saved widget state (`st.session_state._saved_*`) because they bypass form submit.
- CSS for layout: don't use custom HTML wrappers around Streamlit widgets — use `data-testid` selectors.
