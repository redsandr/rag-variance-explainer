## Problem

Before Phase 1, the RAG pipeline:
- Generated free-text answers with post-hoc regex verification (`verify_answer_llm()`)
- Had no structural contract between LLM and downstream code
- Could not enforce quote-level grounding at generation time
- `verify_answer_llm()` appended warning text to answers, poisoning faithfulness eval

## Solution: 3-Layer Structured Output

### Layer 1 — Pydantic Schema (`src/verifier.py`)

```python
class VerifiedClaim(BaseModel):
    statement: str
    source_chunk_index: int
    source_quote: str

class StructuredAnswer(BaseModel):
    answer: str
    claims: list[VerifiedClaim]
    can_answer: bool
    confidence: float = Field(ge=0.0, le=1.0)
```

- `resolve_refs()` — inlines `$defs` to work around llama.cpp grammar bug (#21228)
- `parse_answer()` — 3-layer JSON parsing (full → partial → None)
- `validate_answer()` — checks every `source_quote` exists verbatim in its chunk
- `fallback_answer()` — graceful refusal when structured parsing fails

### Layer 2 — Prompt Injection (`src/prompts.py`)

```python
def build_rag_system_prompt(structured: bool = True) -> str:
    schema = get_structured_schema()
    fmt = build_format_prompt(schema)
    return SYSTEM_PROMPT_RAG + fmt
```

- Injects resolved JSON schema + format instruction into system prompt
- `"You MUST return ONLY valid JSON matching the schema below."`
- Field descriptions guide model output

### Layer 3 — Parse → Validate → Fallback (`src/rag.py`)

```
LLM generate → parse JSON → validate quotes verbatim
  ✅ → return {structured: True, claims: [...], confidence: N}
  ❌ → fallback_answer() → free-text answer + log warning
```

Fallback layers (never crashes):
1. `StructuredAnswer.model_validate_json()` — full parse
2. `json.loads()` partial recovery → `can_answer=False`
3. Raw text preservation → `structured_failure_reason: "parse_error"`
4. Graceful "cannot answer" message

### Config (`src/config.py`)

- `RAG_STRUCTURED_OUTPUT_ENABLED` — toggle (default `true`)
- Can disable via `.env` for debugging

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| Path B: prompt injection (not grammar) | Works across all 3 backends (llama.cpp, Anthropic, OpenAI); sidesteps llama.cpp `$ref` bug |
| Flat schema (2 levels) | Avoids grammar rule explosion; 7B models follow flat schemas better |
| Verbatim quote check | Grounding by construction — not "please don't hallucinate" |
| Fallback never crashes | Zero risk; returns free-text if structured fails |
