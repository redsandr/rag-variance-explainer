## 🔴 KRITIS (fix immediately)

### 1. Embedding fallback dimension mismatch

| File | Baris |
|------|-------|
| `src/embedding.py` | 13, 28 |

**Problem:** `_MODEL_NAME` = `nomic-embed-text-v1.5` (768-dim). `_FALLBACK_MODEL` = `all-MiniLM-L6-v2` (384-dim). ChromaDB collection dibuat dengan 768 dimensi. Jika nomic gagal load → fallback digunakan → ChromaDB **crash** saat upsert karena dimensi berbeda.

**Fix:** Raise error jelas atau fallback ke model 768-dim lain.

**Risk:** Index rebuild gagal silent, data corrupted.

---

### 2. BM25 cache stale setelah index rebuild

| File | Baris |
|------|-------|
| `src/retrieval.py` | 184-202 |

**Problem:** `_BM25_CACHE` tidak pernah di-invalidate. Jika `build_index.py` nambah chunk baru → cache tetap serve index BM25 yang basi → retrieval missing chunks.

**Fix:** Cache key include collection hash/version, atau flush on rebuild.

**Risk:** Query setelah rebuild dapet hasil hilang — recall turun tanpa disadari.

---

### 3. `.format()` crash saat jawaban/context mengandung curly braces

| File | Baris |
|------|-------|
| `src/post_process.py` | 185-200 |

**Problem:** `VERIFY_NUMBERS_PROMPT.format(context=context, answer=answer)` — jika answer atau context punya `{...}` literal, Python `str.format()` raise `KeyError`. Catch exception hanya di luar blok (`rag.py:109`) jadi crash mungkin terjadi sebelum catch.

**Status:** **SUDAH FIXED** di prompt template (escaping `{{` `}}`), tapi perlu test untuk konfirmasi.

## 🟡 TINGGI

### 4. POST endpoint menggunakan Query() bukan Body()

| File | Baris |
|------|-------|
| `src/api.py` | 120-125, 161-166 |

**Problem:** `/search` dan `/answer` deklarasi parameter `Query(...)` di HTTP POST. Banyak HTTP client gak kirim query params dengan POST — mereka kirim JSON body. Hasil: parameter hilang, return error.

**Fix:** Ganti `Query()` → `Body()`.

---

### 5. Race condition — global model loading

| File | Baris |
|------|-------|
| `src/embedding.py` | 17-30 |
| `src/cross_encoder.py` | 12-22 |

**Problem:** Singleton pattern tanpa lock. Dua thread bisa lewat guard `if _model is None` bersamaan → model load 2× → resource leak.

**Fix:** Tambah `threading.Lock()`.

---

### 6. BM25 full corpus scan O(n)

| File | Baris |
|------|-------|
| `src/retrieval.py` | 212-218 |

**Problem:** `_retrieve_bm25` score **semua** dokumen di collection (1079 chunks) lalu sort. Tidak seperti dense retrieval (pakai ANN HNSW ChromaDB), BM25 = full scan setiap query. Scalability bottleneck.

**Fix:** Implementasi BM25 approximate (tiered retrieval) atau batasi candidate pool.

---

### 7. Timeout double-counted

| File | Baris |
|------|-------|
| `src/llm.py` | 168, 185 |

**Problem:** `_llm_semaphore.acquire(timeout=_LLM_TIMEOUT)` DAN `future.result(timeout=_LLM_TIMEOUT)` pakai TIMEOUT YANG SAMA. Total wait bisa 2× timeout = 240s (default 120s).

**Fix:** Bagi timeout: semaphore pakai `_LLM_TIMEOUT - elapsed`, atau batalkan salah satu.

---

### 8. MCP tool return error sebagai success

| File | Baris |
|------|-------|
| `src/server.py` | 131-133, 163-165 |

**Problem:** `except Exception as exc: return f"Error: {exc}"` — error dikembalikan sebagai string sukses. AI agent gak bisa bedain hasil vs error.

**Fix:** Return `ToolError` (MCP SDK) atau raise exception.

---

### 9. Prompt separator single newline

| File | Baris |
|------|-------|
| `src/prompts.py` | 94, 106 |

**Problem:** `chr(10)+chr(10).join(context_blocks)` menghasilkan `"\n" + "\n".join(blocks)` = NEWLINE TUNGGAL antar blok. Semua source chunks dempet jadi satu paragraf — harder for LLM to parse.

**Fix:** `"\n\n".join(context_blocks)`.

---

### 10. Chunking overlap encode O(n²)

| File | Baris |
|------|-------|
| `src/chunking.py` | 93-98 |

**Problem:** Untuk 50 chunk, overlap loop panggil `_ENCODING.encode(chunks[i-1])` sebanyak 49×. Padahal token dari chunk sebelumnya udah available — tinggal reuse.

**Fix:** Simpan token hasil encode dari iterasi sebelumnya.
